1. 项目整体在做什么
这个项目是在做“装配调度优化”的论文实验。

它的核心问题是：有一批装配工序，这些工序有前后依赖关系、需要不同站位/班组/工人来完成，还可能出现扰动。项目用几种方法来比较调度效果：

规则法
遗传算法 GA
多智能体强化学习 QMIX
目标不是做产品，而是做实验对比，看不同方法在不同数据集上，谁能把节拍、平滑指数、完成时间这些指标做得更好。

可以把它理解成这条主线：

工序约束Excel -> 参数解析 -> 装配仿真环境 -> 调度策略(规则/GA/QMIX) -> 输出节拍与性能结果

2. 主要目录和模块分别负责什么

第三章
这是数据区，不是主代码。
源数据/数据集数据：原始数据集，比如 工序约束_50.xlsx、工序约束_10010.xlsx
中间数据/训练中间数据：训练日志、依赖说明
中间数据/训练过程图表：训练过程图表
结果数据/实验结果表：最终实验结果
第三章程序
这是主代码区，按实验规模分成 4 套版本。
dalunwen2：50 工序，小规模，偏早期版本
dalunwen2_2：50 工序，数据集2-4、含扰动实验最常用
dalunwen2_3：100 工序版本
第三章air0417：3182 工序大规模版本
每套代码里同名模块基本职责一致
parameter.py：读 Excel，生成工序前驱/后继、工时、初始可执行工序等参数
config.py：训练开关、超参数、是否加载模型
NN.py：神经网络定义
policy.py：QMIX 训练核心
agent.py：智能体动作选择与训练封装
rollout*.py / onlinerollout*.py：装配仿真环境和 episode 生成
QMIX*.py / onlineqmix.py：训练入口、在线应用入口
GA_rules.py / GA_dis.py / Fitness*.py：遗传算法和适应度评估
rules.py / simulation.py / simplesimulation.py：规则法或基线仿真

4. 主流程怎么走

先读数据集
从 parameter.py 开始，把某个 工序约束_*.xlsx 读进来，得到：
每道工序的前驱工序
后继工序
工时
初始可执行工序
一些后续计算优先级要用的特征
构建仿真环境
rollout*.py 或 simulation*.py 里定义：
站位 Station
班组/工人 Team / Worker
飞机或工件对象
工序完成触发关系
这里用 simpy 让装配过程真正“跑起来”。
选策略
不同入口文件会接上不同策略：
rules.py：按固定规则分配工序和班组
GA_rules.py / GA_dis.py：用遗传算法搜索更优规则参数
QMIX*.py：用强化学习训练策略
onlineqmix.py：加载已有模型直接应用
执行一个或多个 episode
每次 episode 本质上就是“给定当前节拍和规则/动作，跑完整个装配过程”。
输出结果
结果通常包括：
节拍变化
平滑指数
完工时间
奖励值
模型参数文件 .pkl
中间或最终 Excel 表
