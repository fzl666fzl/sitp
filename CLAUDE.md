# CLAUDE

## Skills Guidance for GitHub Copilot by Andrej Karpathy

1. **Understand the Problem**: Clearly define the problem you need to solve.
2. **Provide Context**: Include relevant context in your comments for Copilot to generate accurate code.
3. **Be Specific**: Ask for specific functions or implementations rather than vague instructions.
4. **Iterate**: Generate code and then refine it by adjusting your prompts.
5. **Utilize Examples**: Provide example inputs and expected outputs to guide Copilot.
6. **Read the Documentation**: Look up functions, libraries, and API documentation for better prompts.
7. **Experiment**: Don't hesitate to try different variants of comments or code to explore Copilot’s capabilities.
8. **Follow Best Practices**: Incorporate standard coding practices for clarity and maintainability.
9. **Use Comments**: Utilize comments to help Copilot understand the flow and logic of your code.
10. **Review and Test**: Always review generated code and test it thoroughly before use.
